Evaluating the SciFlow framework was performed under the following criteria;

  • Running SciFlow on three different Parsl configurations
    • Threads on a local machine
    • Utilizing cores of a local machine
    • On a cluster
  • Removing the Golang control thread from the framework, while leaving Parsl. The individual python modules were called from a shell script. With Parsl left, we evaluated again with the same three configurations mentioned above.
  • Removing implicit parallelism (Parsl) from the framework and running the Go control thread.
  • Removing both Parsl and Go. The individual modules were run using a shell script.

We ran the above scenarios on a cluster consisting of 4 machines; three machines with 4GB RAM and one machine with 12GB RAM. The dataset used, consists of three months of GDELT data.
