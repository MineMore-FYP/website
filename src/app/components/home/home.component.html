Reusable modules

The modules you create are fully reusable. They can easily be plugged to any workflow you create thereafter.

Spawn multiple instances of the same module in parallel

With a simple ‘for’ loop, Parsl would generate and execute many instances of a single module.

Construct Dynamic Workflows

The Go thread would make a decision regarding the most appropriate path to take and redirect along this path.

Embedded Control Logic

No need to rely on external modules to make run-time decision. Allow SciFlow’s Go Lang control thread to do that for you!

Flexibility

Each module (event) is executed once the event is ready. Channels provide the ability for the workflow to be event-driven.

Composability

A very generic control thread! With minor changes, you can easily construct any workflow.

Predefined workflow instructions

Setup the control thread beforehand and the script would run with no intervention required.

Easy Configuration on HPC

With the use of Parsl, you could easily adjust machine(threads/cores) or cluster configurations, by simply loading a pre-written configuration file.

Map-reduce like approach

The framework’s approach maps to a manual map-reduce-like approach, thereby, providing all the benefits of map reduce as well.
